Was sind Frameworks?

Standardisierte Codebibliothek von zusammengefassten Packets mit Files und Ordnern
  • Fundament für Enwicklung und Gestaltung einer Website
  • Sollen eine einheitliche Struktur für Entwickler bilden
  • Man muss nicht komplett neuanfangen
  • Somit kann man immer wieder auf Codes zurückgreifen

    • Typische Komponenten sind:

    • Standard CSS-Klassen auf denen aufgebaut werden kann
    • Für alle HTML-Elemente definierte Typographie
    • javaScript Bibliotheken
    • Gridsystem auf Basis von CSS
    • Browserkompabilität für aktuellen und älteren Versionen

    Welche Frameworks gibt es?

    Einige Beispiele von einer großen Anzahl an Frameworks:

    • SASS/SCSS
    • YAML 4
    • 960 Grid System
    • Blueprint


    Welches Framework ist das richtige für mich?

    Die Frameworks sind im Grundumfang und Funktionalität sehr ähnlich, d.h. für einen:

  • Persönliche Vorliebe
  • Aktualität
  • Wiederverwertbarkeit
  • Community&Support
  • "Automatische" oder "Manuelle" Erstellung

    • Vorteile

    • Zugriff auf eine große Codebibliothek und somit schnellere Webentwicklung
    • Für zukünftige Webseiten wiederverwendbar
    • Sauberes und übersichtliches Layout
    • Übergreifende Browserkompabilitäten
    • Regelmäßige Updates
    • Zahlreiche Communities und Foren für Austausch von Erfahrungsberichten und Hilfen


    Nachteile

    • Flache Lernkurve für CSS-Anfänger
    • Überflüssige extra Codes, die wahrscheinlich nicht gebraucht werden und den Quelltext unübersichtlicher machen
    • Können fehlerhafte Codes von den Entwicklern enthalten
    • Gezwungen die Framework Semantik zu lernen

    Das Framework SASS / SCSS

    Auf Ruby basierende Stylesheet-Sprache, welche in CSS interpretiert wird.
    Von Hampton Catlin und Nathan Weizenbaum entwickelt.

    Es gibt zwei Syntaxen:

      SASS - Syntactically Awesome Stylesheet
      SCSS - Sassy Cascading Stylesheet


  • Für Gebrauch am eigenen Rechner muss Ruby installiert sein

    • Wie setze ich SASS / SCSS ein?

  • Über das Terminal
  • Mittels der App Scout

    Variablen

    Vereinfacht das Verändern von z.B. Farben, Font-Styles oder anderen CSS Werten.
    Mit dem $-Zeichen wird eine Variable deklariert und aufgerufen.

    CSS

    * {
  margin: 0px;
  padding: 0px;
}

body {
  background-color: black;
}

.big {
  height: 280px;
  width: 280px;
}

.little {
  height: 50px;
  width: 50px;
}

.red {
  border: 5px ridge #be1622;
  background-color: #e6332a;
}

.blue {
  border: 5px ridge #1d71b8;
  background-color: #36a9e1;
}

.green {
  border: 5px ridge #008d36;
  background-color: #009640;
}

.yellow {
  border: 5px ridge #e94e1b;
  background-color: #f39200;
}

.flexibel {
  position: absolute;
  left: 50%;
}

    SCSS

    $border : 5px;
$big : 280px;
$small : 50px;

    margin: 0px;
    padding: 0px;
}

    background-color: black;
}

.big {
    height: $big;
    width: $big;
}

.little {
    height: $small;
    width: $small;
}

    background-color: #e6332a;
}

.blue {

    Mixin

    Mit Mixin kann man eigene Vorlagen definieren und später in der eigenen Datei
    beliebig oft aufrufen, ohne sie jedes mal einzeln zu ändern.

    CSS

    @mixin box($size:40px) {
	width: $size;
	height: $size;
}
.box {
	@include box(100px);
	background: #fc0;
}

    SCSS

    .box {
  width: 100px;
  height: 100px;
  background: #fc0;
}

    Nesting

    Nesting ermöglicht es eine visuelle Hierarchie ähnlich wie bei HTML in CSS zu erstellen.
    Es erleichtert das Lesen und Organisieren der eigenen CSS.

    CSS

    * {
  margin: 0px;
  padding: 0px;
}

body {
  background-color: black;
  color: yellow;
}

nav {
  margin-top: 50px;
  background-color: white;
  color: red;
}
nav ul {
  color: green;
}
nav li {
  color: blue;
  display: inline;
}

    SCSS

    $listet: green;
$inline: blue;
$noNav: yellow;
$unlistet: red;
$white: white;
$black: black;



*{
    margin: 0px;
    padding: 0px;
}

body{
    background-color: $black;
    color: $noNav;
    }
nav {
    margin-top: 50px;
    background-color: $white;
    color: $unlistet;
  ul {
      color:$listet;
  }

  li { 
      color:$inline;
      display: inline;
}
}
