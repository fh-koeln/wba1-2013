1 Was ist Git? Und warum nutzen wir es?

  • Ein modernes Versionskontrollsystem, entwickelt von Linus Torvalds
  • Erstmal „nur“ ein Kommandozeilentool
  • Dezentrale „Repositories“

  • Vorteile von Git:

    • Optimiert auf gemeinsames, verteiltes arbeiten
    • Besonders gute Toolunterstützung / 3rd-Party-Apps

2 Was ist GitHub? Und warum nutzen wir es?

  • Eine zentrale Infrastruktur für Git (eine „Autorität“)

    ▸ „ein zentrales Repository“

  • Von der OpenSource-Community und Arbeitgebern akzeptiert

    ▸ „Social Development“

  • GitHub bietet zudem optional:

    • Webinterface für alle wichtigen Funktionen
    • Task-Verwaltung „Issues“
    • Ein Wiki pro Projekt
    • Eine Webseite pro Projekt
    • Hosting...

Was ist ein Branch?

  • branch → Ermöglicht paralleles arbeiten an unterschiedlichen „Features“
  • merge → Bezeichnet das zusammenführen unterschiedlicher branches

4 Konflikte

  • Wann treten Konflikte auf?
  • → Bei Änderungen, die nicht automatisch mergebar sind.
  • → Beim synchonisieren (pull und mergen)
  • Kann man das Risko reduzieren?
  • → Häufig(er) synchonisieren
  • → Mit „Umbauarbeiten“ (etwa neu formatieren) zurückhalten
