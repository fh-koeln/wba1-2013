Warum testen?

  • möglichst fehlerfreie und stabillaufende Software
  • Menschen machen immer Fehler
  • Fehlerwahrscheinlich steigt mit Größe und Komplexität der Anwendung
    • Beispiel: NASA-Rakete Mariner 1

  • Quellcode
  • Funktionalität

  • Sicherheit
  • Bedienbarkeit

Testing-Pyramide

Wie testen?

White-Box-Test

  • Kenntnisse der inneren Programmstruktur bzw. des Quellcodes
  • Testfälle werden anhand der inneren Programmstruktur festgelegt
  • Verschiedene Testmethoden (z.B. Überdeckungstests)
  • Ziel: Quellcode muss gewisse Qualitätskritieren erfüllen (z.B. Zeilenüberdeckung)

Wie testen?

Unittest


Einzelne Module werden getestet

  1. Ausgangszustand initialisieren
  2. die zu testenden Operationen ausführen
  3. Ist-Ergebnis mit Spezifikationen vergleichen

Wie testen?

Black-Box-Test

  • Tester haben keine Kenntnisse der inneren Programmstruktur
  • Mit Fallbeispielen das Verhalten testen
  • Aufwendige Erstellung der Tests
  • Ziel: Übereinstimmung des Softwareverhaltens mit den Spezifikationen

Testen mit Jasmine

Was ist Jasmine?

  • Jasmine ist ein Unit-Testing-Framework für JavaScript
  • Beinhaltet viel aus anderen Tests wie zB. ScrewUnit, JSSpec, JSpec und RSpec
  • Ziele der Entwickler des Jasmine Tests:
    1. Es sollte nicht auf bestimmten Browser, Platformen, etc., abestimmt sein
    2. Es soll Überall funktionieren, wo auch JavaScript läuft (Browsers, Server, Handys, etc.)
    3. Die Syntax ist sprachüblich und einfach zu verstehen und motviert zum Test schreiben

Testen mit Jasmine

Besonderheiten

  • Jasmine setzt nicht auf bestimmte JavaScript Frameworks.
  • Es ist ein BDD Framework
  • Es braucht kein DOM

Testen mit Jasmine

Matcher

  • Jeder Matcher kann mit .not verneint werden
  • Hier ein paar Beispiele für ein paarMatcher:
    • expect(x).toEqual(y); (Test gibt kein Fehler, falls y == x)
    • expect(x).toBe(y); (Test gibt kein Fehler, falls y gleiches Objekt wie x)
    • expect(x).toContain(y); (Test gibt kein Fehler, falls String oder Array x, y beinhaltet)
    • weitere
